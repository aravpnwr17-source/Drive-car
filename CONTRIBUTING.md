# Contributing to Drive-Car

Thank you for your interest in contributing to the Drive-Car project! Here are some guidelines:

## Code of Conduct
- Be respectful and professional
- Welcome all contributors
- Report issues constructively

## How to Contribute

### 1. Fork the Repository
```bash
git clone https://github.com/aravpnwr17-source/Drive-car.git
cd Drive-car
```

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Follow C# naming conventions
- Add XML documentation comments
- Test thoroughly

### 4. Commit with Clear Messages
```bash
git commit -m "Add: description of your feature"
git commit -m "Fix: description of the bug fix"
git commit -m "Docs: description of documentation update"
```

### 5. Push and Create Pull Request
```bash
git push origin feature/your-feature-name
```

## Commit Message Format

- **Add**: New features
- **Fix**: Bug fixes
- **Docs**: Documentation changes
- **Refactor**: Code refactoring
- **Style**: Code style changes
- **Test**: Adding or updating tests
- **Perf**: Performance improvements

## Code Standards

### C# Style Guide
- PascalCase for classes and methods
- camelCase for local variables
- _camelCase for private fields
- UPPER_SNAKE_CASE for constants

### Documentation
```csharp
/// <summary>
/// Brief description of the method
/// </summary>
/// <param name="paramName">Parameter description</param>
/// <returns>Return value description</returns>
public void MethodName(string paramName)
{
    // Implementation
}
```

## Pull Request Process

1. Ensure code follows style guide
2. Update documentation if needed
3. Test on target platform
4. Request review from maintainers
5. Address review feedback

## Reporting Issues

Include:
- Clear title describing the problem
- Detailed description
- Steps to reproduce
- Expected vs actual behavior
- System information

## Questions?

Feel free to open a discussion or reach out to the team!
